
# 核心痛點

## 概要
SS／5ch 影片創作者最大的時間成本，是剪輯，但最爽的部分是想劇本寫劇本，如果我能提供一套工作流，讓創作者寫完劇本的瞬間，就能跳過剪輯，渲染出影片，就能讓創作者爽了又爽。

這是專為 2D 劇場影片打造的 Script-to-Video 引擎。
使用者寫劇本，而不是剪影片。
語法盡量一看就懂 就像真的劇本那樣。

使用者只需要準備素材（一次性）與劇本（每支影片），其餘的字幕、TTS、角色演出與影片渲染皆由引擎自動完成。目標不是取代傳統剪輯軟體，而是讓劇場型影片的製作流程從「剪輯」轉變為「渲染」。

## 工作流比較：傳統 V.S 我提出的方案

### 傳統
```text
想劇本
	↓
生成 TTS
	↓
匯入剪輯軟體
	↓
排字幕
	↓
角色動畫
	↓
背景切換
	↓
調 Timeline
	↓
Render
```

### 我的方案
```text
想劇本
	↓
寫 Script
	↓
Render
```

# MVP
## 流程角度探討
1. 引入模板
2. 設定素材(腳色/背景)
3. 寫腳本
	Editor 提供：

	自動補全
	Token Highlight
	即時錯誤提示
	Hover 預覽素材
	跳轉素材
4. 渲染

### 功能角度探討
## 必要功能

* 專案管理
* 模板
* 素材管理
* 劇本編輯器
* 自動補全
* 即時預覽
* Render MP4

## 不做（至少 MVP 不做）
* 模板CameraWork(比如：抖動攝影機、ZOOM到某個腳色、frame腳色群、BodyShot、UpperBodyShot、CloseUp)
* 簡單特效/濾鏡/畫面效果
* Keyframe 編輯(比如：動畫表情)


## 絕對不做
* Timeline Editor
* 解說影片工作流(因為需要細緻時間控制，這是Timline editor的特長，而不是我的方案的目標)
* MV／AMV 類型剪輯(同上)
* 複雜特效(同上)
* 精細動畫曲線(同上)
* 自由 CameraWork

# App 工作流

## 1. 建立專案

```
新建專案
```

可選：

* 使用劇場模板（推薦）
* 空白專案

---

## 2. 匯入模板

模板包含：

* 角色
* 表情
* 動作
* 背景
* BGM
* SE
* Theme（演出規則）

使用者不需要先建素材。


## 3. 素材設定

沒有模板時，也能建立自己的素材。
像是：
* 表情
* 動作
* 站位
* 背景
* BGM
* Theme


## 4. 劇本編輯

主要畫面，全部鍵盤操作。

例如：

```text
背景 教室 早上
小傑 登場 左
奇犽 登場 右
小傑 「昨天發生一件事。」 表情 認真
奇犽 「真的假的？」 表情 驚訝

```

Editor 提供：

* 自動補全
* Token Highlight
* 即時錯誤提示
* Hover 預覽素材
* 跳轉素材

不用記語法。



## 5. 即時預覽

修改腳本，會即時更新預覽

Renderer 即時播放。




## 6. Render

按 Render，不更新預覽直接渲染影片
