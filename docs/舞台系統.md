# 舞台系統

## 定位

舞台系統是 WakuWaku Drama Club 的演出層。

它採用紙劇場概念：不管角色來源是圖片、3D 模型或 Live2D，對劇本和導演 API 來說都只是舞台上的紙片人。系統可以使用 Node3D、Camera3D、Sprite3D、MeshInstance3D 等 3D 節點實作，但不把真 3D 世界規則暴露給劇本。

核心原則：

1. 高層維持 anchor-first 的舞台語意。
2. 3D 能力只用來取得 camera、transform、renderer 彈性。
3. 角色深度不是敘事規則，最多只作為 layer / sorting / parallax。
4. DialoguePanel 預設是 UI overlay，不是 3D 舞台物件。

## 命名

系統名稱：Stage System / 舞台系統

建議程式命名：

1. StageViewport
2. StageActor
3. StageObject
4. StageAnchor
5. StageCamera
6. IStageActorRenderer
7. ImageActorRenderer
8. ModelActorRenderer
9. Live2DActorRenderer

`Stage` 比 `World` 更適合，因為這不是真 3D 世界；也比 `Renderer` 更適合，因為舞台系統同時包含角色、物件、錨點、鏡頭與動畫。

## 舞台位置

劇本與 instruction 層不應該承諾 2D 或 3D 座標格式，而是使用 StageAnchor 名稱描述位置。

範例：

```text
登場 潔莉諾 在 center
移動 潔莉諾 到 left 在 0.5
```

StageViewport 負責把 anchor 名稱解析成舞台內部位置：

```text
StageAnchor("center") -> Transform3D / Vector3
```

對外不應要求使用者思考 Vector2、Vector3 或 Z 軸。需要排序或前後層次時，優先使用 `Layer`、`SortOrder` 或 `PlaneIndex` 這類語意。

StageActor 的低層 API 可以吃內部位置型別，但 instruction 應該盡量停在 anchor 名稱：

```csharp
AnimationClip Teleport(StageAnchor anchor)
AnimationClip MoveTo(StageAnchor anchor, double duration)
```

若有需要，也可以保留 internal/helper 用的直接座標方法，但不作為劇本語意的主入口。

## StageActor

StageActor 是舞台上的角色槽位，建議繼承 Node3D。

StageActor 負責：

1. 保存角色 id 與 ActorDefinition。
2. 提供紙劇場式動畫 API。
3. 管理 renderer。
4. 接收 StageAnchor 解析後的位置，套用內部 3D transform。

StageActor 對外 API 應該維持簡單：

```csharp
AnimationClip FadeIn()
AnimationClip Teleport(StageAnchor anchor)
AnimationClip MoveTo(StageAnchor anchor, double duration)
AnimationClip MakeExpression(string expression)
AnimationClip TakePose(string pose)
```

## Actor Renderer

Actor renderer 負責產生角色可視內容。不同素材來源可以有不同 renderer，但都遵守同一個紙片人語意。

建議介面：

```csharp
public interface IStageActorRenderer
{
    void Setup(ActorDefinition definition);
    AnimationClip MakeExpression(string expression);
    AnimationClip TakePose(string pose);
    Node3D GetVisualRoot();
}
```

### ImageActorRenderer

ImageActorRenderer 是近期目標。

它使用 Sprite3D 顯示現有 2D 角色圖片，並支援現有的 expression / pose 概念。

### ModelActorRenderer

ModelActorRenderer 是往後目標。

3D 模型不需要先經過 SubViewport 轉成圖片。若模型有骨架或動畫，renderer 可以直接 instantiate PackedScene，使用 MeshInstance3D、Skeleton3D、AnimationPlayer 等節點播放模型動畫。

紙劇場限制不放在渲染技術，而放在對外 API：

1. 不提供真 3D 走位語意。
2. 不把模型深度暴露給劇本。
3. 不提供自由環繞角色的世界鏡頭。
4. 模型動畫是 renderer 內部細節。

### Live2DActorRenderer

Live2DActorRenderer 是往後目標。

它應該和圖片、模型 renderer 一樣，只對 StageActor 提供表情、姿勢、動作等紙片人操作。

## StageCamera

StageCamera 是導演鏡頭，不是玩家在 3D 世界中的相機。

它服務 2D 構圖：

1. Pan
2. Zoom
3. Shake
4. Frame actor
5. BodyShot
6. UpperBodyShot
7. CloseUp

CameraWork 模板可以往後做，但 0.2.0 只需要先讓舞台架構容納 camera 概念。

## DialoguePanel

DialoguePanel 先保留為 Control / UI overlay。

原因：

1. 對話框通常需要穩定貼在畫面上。
2. UI overlay 比 3D 物件更容易處理字體、排版、解析度與輸出。
3. 目前沒有必要讓對話框參與 3D camera transform。

若未來需要讓對話框也成為舞台物件，可以再新增 StageDialoguePanel 或 StageObject，不要在第一版混進 StageActor 遷移。
