# 資源管理

資源管理處理專案目錄中的素材如何被掃描、索引與引用。

核心原則是：

```text
Filesystem is source of truth.
```

使用者可以用繪圖、音訊或其他外部工具直接輸出到專案素材目錄。App 只負責掃描目錄、建立索引、補充無法從檔案結構推斷的 metadata。

## 目錄結構

```text
MyProject/
├── project.json
├── scripts/
│   ├── ep01.waku
│   └── ep02.waku
└── assets/
    └── default/
        ├── actors/
        │   └── mamedou/
        │       ├── meta.json
        │       ├── expressions/
        │       │   ├── normal.png
        │       │   └── happy.png
        │       └── bodies/
        │           └── uniform.png
        ├── backgrounds/
        │   └── classroom_morning.jpg
        └── audio/
            └── question.wav
```

`assets/{pack}/` 是資源命名空間，也可以視為模板包或素材包。

## 命名規則

資源相關的檔案與目錄名稱不能有空格。

不建議：

```text
assets/default/actors/毛豆 01/
assets/default/backgrounds/classroom morning.jpg
```

建議：

```text
assets/default/actors/mamedou_01/
assets/default/backgrounds/classroom_morning.jpg
```

原因：

- 劇本語法目前以空白切 token。
- resource id 會出現在劇本與補全候選中。
- 無空格命名可以避免 parser、completion、路徑顯示與跨平台 shell 操作出現歧義。

顯示名稱可以之後由 `meta.json` 提供，不必依賴檔名本身。

## 資源 ID

主要資源使用 `pack/name`：

```text
default/mamedou
default/classroom_morning
default/question
```

角色子資源使用 actor-local 名稱：

```text
normal
happy
uniform
```

劇本範例：

```text
背景 default/classroom_morning
音效 default/question
default/mamedou 「你好」 表情 happy
```

## 子資源規則

| 層級 | 需要 meta.json？ | 說明 |
|------|----------------|------|
| Actor 目錄 | 是 | 儲存 display name、voice、default expression 等無法推斷的設定 |
| expressions/ 內的圖片 | 否 | 檔名即名稱，存在即有效 |
| bodies/ 內的圖片 | 否 | 檔名即名稱，存在即有效 |
| backgrounds/ 內的圖片 | 否 | 檔名即名稱，存在即有效 |
| audio/ 內的音檔 | 否 | 檔名即名稱，存在即有效 |

`meta.json` 只記錄無法從檔案結構推斷的 metadata，不記錄子資源路徑。

## Actor Metadata

Actor 目錄的 `meta.json` 用來保存純量設定：

```json
{
  "display_name": "毛豆",
  "default_expression": "normal",
  "default_body": "uniform",
  "voice": {
    "source": "debug",
    "voice_id": "default"
  }
}
```

子資源仍由 filesystem 掃描取得。

## 掃描

掃描發生在：

- App 啟動並載入專案後。
- 使用者手動 Refresh。
- 未來的素材目錄輔助流程完成後。

掃描規則：

- 掃描 `assets/` 底下所有 pack。
- 掃描 `actors/`、`backgrounds/`、`audio/`。
- 發現 actor 目錄沒有 `meta.json` 時，自動建立預設檔。
- 發現 expressions / bodies 時，列舉圖片檔。
- 不把掃描結果寫入 `project.json`。

## 不做 Import Copy Workflow

不做核心匯入流程：

```text
選擇外部檔案
  -> App 複製到 assets/
  -> 使用者修改原檔後需要重新匯入
```

這會破壞素材反覆編輯的工作流。

推薦工作流是：

```text
外部創作工具
  -> 直接輸出到專案 assets 目錄
  -> App Refresh 掃描
  -> 劇本與補全立刻可用
```

未來可以做的是素材目錄輔助：

- 建立角色/背景/音效的標準資料夾。
- 開啟素材資料夾。
- Refresh 掃描。
- 顯示掃描結果與缺失警告。

## 原則

1. Filesystem 是素材 source of truth。
2. `ResourceManager` 掃描 filesystem 並建立 runtime index。
3. `project.json` 不保存掃描到的子資源列表。
4. `meta.json` 只補充 filesystem 表達不了的資訊。
5. 子資源路徑永遠靠掃描取得。
