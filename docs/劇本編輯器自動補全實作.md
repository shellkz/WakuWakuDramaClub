# 劇本編輯器自動補全實作

這份文件描述自動補全系統的實作藍圖。
需求與使用者體驗請參考 `docs/劇本編輯器自動補全.md`。

## 實作階段

```text
0. Instruction schema 前置作業 [done]
   定義 instruction / statement / argument schema。
   讓 Instruction 的 BindData 與 completion 共用同一批 declarative binding definitions。
  
0. 建立 InstructionRegistry [done]

  InstructionRegistry
    byId -> dict[string, InstructionRegistryEntry]
    byKeyword -> dict[string, InstructionRegistryEntry]

  InstructionRegistryEntry
    Id
    Keyword
    Type
    Schema

  設計成普通 service object，不做 Godot autoload。


  由上層建立 / 持有 registry，並注入需要的模組：
    ScriptParser
    CompletionAnalyzer
    CompletionProvider

  Most logic can be stripped from ScriptParser

1. Parser 基礎建設整理 [done]
   暴露 tokenizer、keyword 判斷、statement grouping、actor shorthand normalize。
   parser 與 completion analyzer 共用這些基礎邏輯。

2. Completion line model
   定義 CompletionLineResult、CompletionRawInstruction、CompletionStatement、CompletionCursorContext。
   這些資料結構要能描述不完整的目前行。

3. CompletionAnalyzer
   input: current line + caret column
   output: CompletionLineResult
   使用 InstructionRegistry 解讀目前行與游標所在 slot。

4. CompletionProvider
   input: CompletionLineResult
   output: CompletionOption list
   使用 InstructionRegistry 與 ResourceManager 產生候選。

5. CodeEdit integration
   EditingMenu / ScriptEditor 只接 CodeEdit API。
   TextChanged 時用 analyzer/provider 判斷是否 RequestCodeCompletion。
   CodeCompletionRequested 時把 provider options 塞進 CodeEdit。

6. 移除舊 completion 實作
   移除舊的淺層 CompletionContext enum。
   移除 TryGetKeywordRemainder / TryGetActorInstructionRemainder 這類 UI 層字串判斷。

7. 後續能力
   錯誤提示、hover、跳轉素材、自訂 popup/list UI。
```

## Instruction Schema

Instruction schema 描述 instruction 的語法結構，不描述 instruction 如何執行。

它需要回答：

```text
有哪些 instruction。
instruction 的 primary statement 是什麼。
primary statement 需要哪些 arguments。
instruction 支援哪些 option statements。
option statement 需要哪些 arguments。
每個 argument 是哪種 ValueKind。
```

### ValueKind

`ValueKind` 描述 argument 的語意與候選來源。

```text
Actor
DialogueText
Background
Audio
ActorExpression
ActorPose
Anchor
Float
Int
String
```

資源型 value 由 `ResourceManager` 提供候選。
資料型 value 通常用於 validation，或提供固定候選。

### ArgumentBindingDefinition

描述一個 statement argument。

```text
ArgumentBindingDefinition
  Name: string
  ValueKind: CompletionValueKind
  Required: bool
  DefaultValue: string?
```

主要用途：

```text
CompletionAnalyzer
  用它判斷 caret 正在填哪種 argument。

CompletionProvider
  用它決定候選來源。

Instruction.BindData
  用它從 RawInstruction 取值。
```

### StatementBindingDefinition

描述一個 statement。

```text
StatementBindingDefinition
  Name: string
  Arguments: List<ArgumentBindingDefinition>
  AllowMultiple: bool
```

第一版可以先假設 `AllowMultiple = false`，因為目前正式 `RawInstruction.Statements` 是 dictionary，同名 statement 不會保留多份。

### InstructionBindingDefinition

描述一個 instruction。

```text
InstructionBindingDefinition
  InstructionName: string
  PrimaryStatement: StatementBindingDefinition
  OptionStatements: List<StatementBindingDefinition>
```

規則：

```text
InstructionName 通常等於 PrimaryStatement.Name。
PrimaryStatement 決定 RawInstruction.Type。
OptionStatements 是這個 instruction 支援的修飾 statement。
```

範例：

```text
Instruction: 對話
PrimaryStatement:
  對話
    actor: Actor
    speech: DialogueText
OptionStatements:
  表情
    expression: ActorExpression
  動作
    pose: ActorPose
```

```text
Instruction: 移動
PrimaryStatement:
  移動
    actor: Actor
OptionStatements:
  到
    destination: Anchor
  在
    duration: Float
```

### Instruction

`Instruction` 子類別應定義自己的 binding definitions。

```text
Instruction
  GetName(): string
  GetBindingDefinition(): InstructionBindingDefinition
  BindData(raw: RawInstruction): void
  BakeAsAnimation(...): Task<AnimationPack>
```

實作原則：

```text
schema 和 BindData 共用同一批 declarative binding definitions。
不要從 BindData 執行結果反推 schema。
BindData 只負責 runtime 欄位綁定。
```

概念上，`BindData` 可以透過 helper 使用 definition 取值：

```text
BindData(raw)
  Actor = raw.GetArgument(definition, "actor")
  Destination = raw.GetArgument(definition, "destination")
```

## InstructionRegistry

集中管理 instruction type 與 schema。

```text
InstructionRegistry
  Factories: Dictionary<string, Type>
  Schemas: Dictionary<string, InstructionBindingDefinition>
```

主要函數：

```text
RegisterAll()
  掃描所有 Instruction 子類別。
  建立 instruction name -> C# type。
  建立 instruction name -> binding definition。

CreateInstruction(raw: RawInstruction): Instruction
  使用 raw.Type 找到對應 type 並建立 instruction。

GetInstructionNames(): IReadOnlyList<string>
  回傳所有 instruction keyword。

GetSchema(instructionName: string): InstructionBindingDefinition?
  查 instruction schema。

GetOptionStatements(instructionName: string): IReadOnlyList<StatementBindingDefinition>
  查某個 instruction 支援的 option statements。

GetAllStatementKeywords(): IReadOnlySet<string>
  收集所有 primary statement 與 option statement keyword。

IsInstructionKeyword(token: string): bool
IsStatementKeyword(token: string): bool
```

`ScriptParser` 應改用 `InstructionRegistry` 取代自己維護 factory 與 hardcoded keywords。

## Parser 基礎建設

completion analyzer 與正式 parser 需要共用底層語法工具。

### ScriptToken

token 要保留位置，completion 才能定位 caret。

```text
ScriptToken
  Type: TokenType
  Value: string
  StartColumn: int
  EndColumn: int
```

`EndColumn` 建議使用 exclusive index。

### ScriptTokenizer

```text
ScriptTokenizer
  TokenizeLine(line: string, registry: InstructionRegistry): List<ScriptToken>
```

責任：

```text
依空白切 token。
保留 token column range。
辨識 quoted dialogue token。
使用 registry 判斷 keyword。
允許不完整 token。
```

第一版可以沿用現有空白分隔規則。
未來若要支援 escape、跨空白 quoted text，再擴充 tokenizer。

### ScriptLineNormalizer

處理 actor shorthand。

```text
ScriptLineNormalizer
  Normalize(tokens: List<ScriptToken>, registry: InstructionRegistry, resources: ResourceManager): NormalizedLine
```

主要情境：

```text
default/毛豆 "台詞"
  -> 對話 default/毛豆 台詞

default/毛豆 登場
  -> 登場 default/毛豆

default/毛豆 移動 到 右
  -> 移動 default/毛豆 到 右
```

`NormalizedLine` 需要保留原始 token 位置與 normalize 後的語法 token 對應，讓 analyzer 能定位 caret。

```text
NormalizedLine
  Tokens: List<ScriptToken>
  IsActorShorthand: bool
  ActorId: string?
```

### StatementGrouper

```text
StatementGrouper
  Group(tokens: List<ScriptToken>): List<CompletionStatement>
```

規則：

```text
第一個 token 開始 primary statement。
後續 keyword token 開始新的 option statement。
非 keyword token 屬於目前 statement 的 argument。
```

## Completion Line Model

line model 必須允許 incomplete 狀態。

### CompletionLineResult

```text
CompletionLineResult
  Raw: CompletionRawInstruction
  Cursor: CompletionCursorContext
```

`Raw` 描述目前行已經能理解成什麼結構。
`Cursor` 描述 caret 正在填哪個 slot。

### CompletionRawInstruction

```text
CompletionRawInstruction
  Type: string?
  Statements: List<CompletionStatement>
  IsActorShorthand: bool
  ActorId: string?
  IsComplete: bool
```

規則：

```text
Type 可能是 null，代表目前還沒確定 instruction。
Statements[0] 是 primary statement。
Statements[1..] 是 option statements。
ActorId 用於 actor shorthand 已經辨識 actor，但尚未決定 action 的狀態。
```

### CompletionStatement

```text
CompletionStatement
  Type: string
  Arguments: List<CompletionArgument>
  StartColumn: int
  EndColumn: int
```

### CompletionArgument

```text
CompletionArgument
  Value: string
  StartColumn: int
  EndColumn: int
```

### CompletionCursorContext

```text
CompletionCursorContext
  Kind: CompletionCursorKind
  Prefix: string
  InstructionName: string?
  StatementName: string?
  StatementIndex: int?
  ArgumentIndex: int?
  ValueKind: CompletionValueKind?
```

`CompletionCursorContext` 是 provider 提供候選的主要依據。

### CompletionCursorKind

```text
PrimaryStart
InstructionArgument
ActorAction
StatementName
StatementArgument
DialogueText
Unknown
```

說明：

```text
PrimaryStart
  行首或第一個 token 尚未確定是 instruction keyword 還是 actor id。

InstructionArgument
  caret 位於 primary statement 的 argument。

ActorAction
  已輸入完整 actor id，但尚未選擇對話 / 登場 / 移動。

StatementName
  caret 位於 option statement keyword。

StatementArgument
  caret 位於 option statement argument。

DialogueText
  caret 位於對話文字中，不提供 option 候選。
```

## CompletionAnalyzer

將目前行與 caret 轉成 `CompletionLineResult`。

```text
CompletionAnalyzer
  Registry: InstructionRegistry
  ResourceManager: ResourceManager
```

`ResourceManager` 只用於辨識 actor shorthand 的 actor id。
候選產生仍交給 `CompletionProvider`。

主要函數：

```text
Analyze(line: string, caretColumn: int): CompletionLineResult
```

內部流程：

```text
1. Tokenize line with positions。
2. Normalize actor shorthand。
3. Group tokens into statements。
4. Build CompletionRawInstruction。
5. Locate caret token / slot。
6. 使用 registry schema 判斷 ValueKind。
7. Build CompletionCursorContext。
```

### Partial 狀態範例

行首輸入一個字：

```text
背[cursor]
```

結果：

```text
Raw:
  Type = null
  Statements = []

Cursor:
  Kind = PrimaryStart
  Prefix = 背
  ValueKind = null
```

actor id prefix：

```text
def[cursor]
```

結果：

```text
Raw:
  Type = null
  Statements = []

Cursor:
  Kind = PrimaryStart
  Prefix = def
```

actor 已確定但 action 未確定：

```text
default/毛豆 [cursor]
```

結果：

```text
Raw:
  Type = null
  IsActorShorthand = true
  ActorId = default/毛豆

Cursor:
  Kind = ActorAction
  Prefix = ""
```

statement argument：

```text
default/毛豆 移動 到 [cursor]
```

結果：

```text
Raw:
  Type = 移動
  Statements[0] = 移動 [default/毛豆]
  Statements[1] = 到 []

Cursor:
  Kind = StatementArgument
  InstructionName = 移動
  StatementName = 到
  StatementIndex = 1
  ArgumentIndex = 0
  ValueKind = Anchor
  Prefix = ""
```

## CompletionProvider

根據 `CompletionLineResult` 產生 options。

```text
CompletionProvider
  Registry: InstructionRegistry
  ResourceManager: ResourceManager
```

主要函數：

```text
GetOptions(result: CompletionLineResult): IReadOnlyList<CompletionOption>
HasOptions(result: CompletionLineResult): bool
```

`HasOptions` 可以先直接呼叫 `GetOptions(result).Count > 0`。

### CompletionOption

```text
CompletionOption
  DisplayText: string
  InsertText: string
  Kind: CompletionOptionKind
  ValueKind: CompletionValueKind?
  MoveCaretLeftBy: int
```

`MoveCaretLeftBy` 用於選擇對話時插入 `""` 後，將 caret 移到兩個引號中間。

### CompletionOptionKind

```text
Instruction
Resource
Statement
Literal
Snippet
```

### 候選規則

```text
PrimaryStart
  instruction keywords + actor ids

ActorAction
  對話 snippet + actor action instruction keywords

InstructionArgument
  依 ValueKind 提供候選

StatementName
  使用 Registry.GetOptionStatements(instructionName)
  排除已出現且 AllowMultiple = false 的 statements

StatementArgument
  依 ValueKind 提供候選

DialogueText
  回傳空 options

Unknown
  回傳空 options
```

### ValueKind 到候選來源

```text
Actor
  ResourceManager.GetActorIds()

Background
  ResourceManager.GetBackgroundIds()

Audio
  ResourceManager.GetAudioIds()

ActorExpression
  從 CompletionLineResult.Raw 找出 primary actor argument。
  ResourceManager 依 actor id 回傳 expressions。

ActorPose
  從 CompletionLineResult.Raw 找出 primary actor argument。
  ResourceManager 依 actor id 回傳 poses。

Anchor
  固定候選：左 / 中 / 右

Float
  第一版可不提供候選，只保留 validation 空間。

DialogueText
  不提供候選。
```

## CodeEdit Integration

UI 層只負責事件與顯示，不負責語法判斷。

```text
EditingMenu
  Analyzer: CompletionAnalyzer
  Provider: CompletionProvider
```

主要函數：

```text
OnScriptChanged()
  if ShouldRequestCompletion():
    ScriptEditor.RequestCodeCompletion(true)

ShouldRequestCompletion(): bool
  result = Analyzer.Analyze(currentLine, caretColumn)
  options = Provider.GetOptions(result)
  return options.Count > 0

OnCodeCompletionRequested()
  result = Analyzer.Analyze(currentLine, caretColumn)
  options = Provider.GetOptions(result)
  foreach option:
    ScriptEditor.AddCodeCompletionOption(...)
  ScriptEditor.UpdateCodeCompletionOptions(true)

OnCodeCompletionOptionConfirmed(insertText, displayText)
  找到對應 option 的 post insert 行為。
  例如 MoveCaretLeftBy = 1 時，將 caret 往左移一格。
```

第一版可以不 cache analyzer/provider 結果。
若同一行同一 caret column 被分析兩次，結果必須 deterministic。

## 舊實作移除

新架構導入後，以下概念應移除或降到相容層：

```text
EditingMenu.CompletionContext enum
TryGetKeywordRemainder(...)
TryGetActorInstructionRemainder(...)
EditingMenu 內直接判斷 Background / Audio / Actor 的字串 if-else
ScriptParser.GetSupportedKeywords() 內 hardcoded 在 / 到 / 表情 / 動作
ScriptParser 自己維護 instruction factory
```

替代關係：

```text
CompletionContext enum
  -> CompletionCursorContext

TryGetKeywordRemainder / TryGetActorInstructionRemainder
  -> CompletionAnalyzer

EditingMenu 候選 if-else
  -> CompletionProvider

ScriptParser factory / supported keywords
  -> InstructionRegistry
```

## 模組依賴

```text
Instruction
  -> InstructionBindingDefinition

InstructionRegistry
  -> Instruction
  -> InstructionBindingDefinition

ScriptParser
  -> InstructionRegistry
  -> ScriptTokenizer
  -> StatementGrouper

CompletionAnalyzer
  -> InstructionRegistry
  -> ScriptTokenizer
  -> ScriptLineNormalizer
  -> StatementGrouper
  -> ResourceManager

CompletionProvider
  -> InstructionRegistry
  -> ResourceManager

EditingMenu / ScriptEditor
  -> CompletionAnalyzer
  -> CompletionProvider
```
