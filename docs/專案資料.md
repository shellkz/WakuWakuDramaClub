# 專案資料

專案資料處理 App 如何保存專案層級設定與工作狀態。它和資源管理不同：資源管理負責掃描 `assets/`，專案資料負責保存 `project.json` 與 runtime session。

## 專案檔案

每個專案根目錄包含一個 `project.json`：

```text
MyProject/
├── project.json
├── scripts/
└── assets/
```

`project.json` 保存專案層級資料，不保存所有掃描到的素材清單。

## Persistent Data

Persistent data 是需要寫入磁碟的專案狀態。

目前概念：

```text
ProjectData
  project_name
  last_opened_datetime
```

未來可加入：

```text
ProjectData
  project_name
  last_opened_datetime
  current_script
  video_profile
  script_language
```

原則：

- 專案層級設定放在 `project.json`。
- 素材子資源不放進 `project.json`。
- 素材 metadata 放在各自目錄的 `meta.json`。
- 劇本內容放在 `scripts/` 的 `.waku` 檔案。

## Runtime State

Runtime state 是 App 執行期間才需要的暫態資料，不寫入 `project.json`，除非未來明確要保存。

例如：

```text
WorkingDirectory
undo stack
current selection
current caret
preview playback state
```

這些資料可以由 UI 或 session 物件持有。

## ProjectSession

`ProjectSession` 持有`Runtime State`跟`ProjectData`，負責目前開啟的專案：

```text
ProjectSession
  // persistant
  ProjectData
  // runtime
  WorkingDirectory
  

  // save/load project
  CreateProject(...)
  Load(...)
  Save(...)
  Close(...)
```

載入專案後，其他系統可以依附專案狀態工作：

```text
ProjectSession.Load(...)
  -> ProjectLoaded
  -> ResourceManager.Refresh()
```

## 與資源管理的邊界

資源管理：

```text
assets/
  pack/
    actors/
    backgrounds/
    audio/
```

專案資料：

```text
project.json
scripts/
runtime session state
project-level settings
```

邊界規則：

- `ResourceManager` 可以讀取目前 `ProjectSession.WorkingDirectory`。
- `ResourceManager` 不應把掃描到的素材清單寫回 `project.json`。
- `ProjectData` 不應保存每個 expression / body / background / audio 的路徑。
- 如果資料可從 filesystem 推斷，就不放進 `project.json`。

## 原則

1. `project.json` 保存專案層級設定。
2. `assets/` 保存素材與素材 metadata。
3. `scripts/` 保存劇本。
4. Runtime state 留在 session / editor，不預設持久化。
5. 只有無法從 filesystem 推斷、且屬於專案層級的資訊才進 `project.json`。
